📋 The Problem

The AI document compliance rating system was producing inaccurate scores:

  • Resumes were rated 95% — non-compliance documents were scoring as "Mature" compliance evidence
  • Real compliance policies were rated 15% — legitimate NDIS compliance documents were marked as "Critical gaps"
  • No visibility into scoring logic — reviewers couldn't understand why a score was assigned
  • No organizational tuning — one-size-fits-all thresholds didn't match different organization cultures

Root cause: The scoring system used keyword-only analysis (looking for words like "must," "reviewed," "documented") without checking if those words appeared in relevant compliance contexts.

⚙️ How the New Scoring Works

The system uses a multi-rubric scoring approach that adapts based on document type:

  • Policy/Procedure: Checks action language (must, shall, will), responsibilities, procedures, timeframes
  • Clinical Reference: Validates credible source, currency, NDIS relevance
  • Implementation Record: Confirms actual data entries, participant identifiers, dates
  • Service Agreement: Verifies participant rights, signatures, service scope
  • Training Material: Assesses learning objectives, NDIS topics, currency

Score Mapping:

Score Status Meaning
≥75% Mature Strong compliance evidence with documented practices
55%–74% OK Acceptable controls with some gaps to address
30%–54% High Risk Gap Significant compliance gaps requiring action
<30% Critical Gap Critical compliance deficiency or not applicable

Guardrail How It Works Impact
Domain Anchor Gating Checks for 15+ compliance keywords (ndis, participant, consent, policy, etc.) <2 anchors → 55% score penalty
Irrelevant Doc Detection Detects resumes, invoices, finance docs by looking for 3+ type-specific markers Irrelevant doc + <3 anchors → capped at 24% confidence
Template Penalty Identifies blank forms by counting unfilled fields (>12 fields or high marker density) Template-like doc → max score 28% (Critical gap)
Citation Requirements Scores penalized if NDIS citations were not retrieved Mature requires ≥2 citations; lexical-only capped at 80%
Evidence Support Validates that evidence quality matches the status claim Mature without strong anchors/citations downgraded to OK

Every analysis includes a detailed diagnostics breakdown showing:

  • Document Length: Word count of extracted content
  • Query Term Count: Number of search terms extracted from the question
  • Snippet Count: Number of relevant passages found in the document
  • Citation Count: How many NDIS references were retrieved
  • Domain Anchor Hits: Count of compliance-specific keywords detected
  • Template/Irrelevant Flags: Boolean flags if document looks like a template or irrelevant type
  • Retrieval Mode: Whether semantic (embedding) or lexical (keyword) search was used
  • Warnings List: Specific reasons why score was capped or reduced

Why this matters: Reviewers can now see exactly why a document scored the way it did, enabling them to provide targeted feedback ("This is actually relevant — improve the domain anchor detection") rather than just "The score is wrong."

Each organization can define its own scoring thresholds without requiring code changes. Access scoring profile settings in the AI Review workspace (⚙️ icon).

Tunable Parameters:

Parameter Default Range Purpose
mature_min_score 0.75 0.50–0.95 Minimum confidence for "Mature" status
ok_min_score 0.55 0.35–mature_min Minimum confidence for "OK" status
high_risk_min_score 0.30 0.10–ok_min Minimum for "High Risk Gap" (below this is "Critical")
mature_min_anchor_hits 4 2–8 Domain anchors required for Mature status
irrelevant_confidence_cap 0.24 0.10–0.40 Maximum confidence for irrelevant documents

Example Use Cases:

  • Stricter org: Lower mature_min_score to 0.80 and mature_min_anchor_hits to 5
  • Lenient org: Raise irrelevant_confidence_cap to 0.30 if org has higher tolerance
  • Focus org: Increase mature_min_anchor_hits for specialized compliance domains

As reviewers assess documents, they can provide feedback on the AI score using three buttons:

  • False Positive: "AI said OK/Mature but it's really not compliant"
  • False Negative: "AI said Critical/High Risk but it's actually better"
  • Correct: "AI got this one right"
